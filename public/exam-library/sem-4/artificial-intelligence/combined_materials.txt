What is knowledge representation?
‚ÄĘKnowledge representation and reasoning (KR, KRR) is the part of Artificial 
intelligence which concerned with AI agents thinking and how thinking contributes 
to intelligent behavior of agents.
‚ÄĘIt is responsible for representing information about the real world so that a 
computer can understand and can utilize this knowledge to solve the complex real 
world problems such as diagnosis a medical condition or communicating with 
humans in natural language.
‚ÄĘIt is also a way which describes how we can represent knowledge in artificial 
intelligence. Knowledge representation is not just storing data into some database, 
but it also enables an intelligent machine to learn from that knowledge and 
experiences so that it can behave intelligently like a human.

[Page 8]
19/12/23 8
What to Represent?
Following are the kind of knowledge which needs to be represented in AI systems:
‚ÄĘObject: All the facts about objects in our world domain. E.g., Guitars contains 
strings, trumpets are brass instruments.
‚ÄĘEvents: Events are the actions which occur in our world.
‚ÄĘPerformance: It describe behavior which involves knowledge about how to do 
things.
‚ÄĘMeta-knowledge: It is knowledge about what we know.
‚ÄĘFacts: Facts are the truths about the real world and what we represent.
‚ÄĘKnowledge-Base: The central component of the knowledge-based agents is the 
knowledge base. It is represented as KB. The Knowledgebase is a group of the 
Sentences (Here, sentences are used as a technical term and not identical with the 
English language).

Approaches to knowledge Representation
‚ÄĘRepresentational adequacy the ability to represent all of the kinds of 
knowledge that are needed in that domain.
‚ÄĘInferential Adequacy: - the ability to manipulate the representation structures 
in such a way as to derive new structures corresponding to new knowledge 
inferred from ol.
‚ÄĘInferential Efficiency: - the ability to incorporate into the knowledge structure 
additional information that can be used to focus the attention of the 
inference mechanism in the most promising directions.
‚ÄĘAcquisitioned Efficiency: - the ability to acquire new information easily. The 
simplest case involves direct insertion by a person of new knowledge into the 
database.

A KNOWLEDGE-BASED AGENT
12
‚ÄĘA knowledge-based agent includes a knowledge base and an inference system.
‚ÄĘA knowledge base is a set of representations of facts of the world. 
‚ÄĘEach individual representation is called a sentence. 
‚ÄĘThe sentences are expressed in a knowledge representation language. 
‚ÄĘThe agent operates as follows: 
1. It TELLs the knowledge base what it perceives. 
2. It ASKs the knowledge base what action it should perform. 
3. It performs the chosen action.

SUMMARY OF KNOWLEDGE BASED AGENTS
27
‚ÄĘIntelligent agents need knowledge about the world for making good decisions. 
‚ÄĘThe knowledge of an agent is stored in a knowledge base in the form of sentences in a 
knowledge representation language. 
‚ÄĘ A knowledge-based agent needs a knowledge base and an inference mechanism. It 
operates by storing sentences in its knowledge base, inferring new sentences with the 
inference mechanism, and using them to deduce which actions to take. 
‚ÄĘA representation language is defined by its syntax and semantics, which specify the 
structure of sentences and how they relate to the facts of the world. 
‚ÄĘThe interpretation of a sentence is the fact to which it refers. If this fact is part of the 
actual world, then the sentence is true.

Truth tables
‚ÄĘLogic, like arithmetic, has operators, which apply to one, two, or more 
values (operands)
‚ÄĘA truth table lists the results for each possible arrangement of operands
‚ÄĘOrder is important: x op y may or may not give the same result as y op x
‚ÄĘThe rows in a truth table list all possible sequences of truth values for n 
operands, and specify a result for each sequence
‚ÄĘHence, there are 2n rows in a truth table for n operands

Unification
‚ÄĘUnification is a ‚Äúpattern-matching‚ÄĚ procedure 
‚ÄĘTakes two atomic sentences, called literals, as input
‚ÄĘReturns ‚ÄúFailure‚ÄĚ if they do not match and a substitution list, őł, if they do
‚ÄĘThat is, unify(p,q) = őł means subst(őł, p) = subst(őł, q) for two atomic sentences, p and q
‚ÄĘőł is called the most general unifier (mgu)

‚ÄĘTo make literals match, replace (universally quantified) variables by terms

Unification algorithm
procedure unify(p, q, őł)
 Scan p and q left-to-right and find the first corresponding
 terms where p and q ‚Äúdisagree‚ÄĚ (i.e., p and q not equal)
 If there is no disagreement, return őł (success!)
 Let r and s be the terms in p and q, respectively,
 where disagreement first occurs
 If variable(r) then {
 Let őł = union(őł, {r/s})
 Return unify(subst(őł, p), subst(őł, q), őł)
 } else if variable(s) then {
 Let őł = union(őł, {s/r})
 Return unify(subst(őł, p), subst(őł, q), őł)
 } else return ‚ÄúFailure‚ÄĚ
 end

Conflict Resolution
‚ÄĘReasoning in a production system can be viewed as a type of

‚ÄĘSelection strategy for rules from the conflict set controls 
search
‚ÄĘProduction system maintains the conflict set as an agenda
‚ÄĘOrdered list of activated rules (those with their conditions

‚ÄĘConflict resolution strategy determines where a

Salience
‚ÄĘRules may be given a precedence order by assigning a

‚ÄĘNewly activated rules are placed in the agenda above all rules 
of lower salience, and below all rules with higher salience
‚ÄĘRule with higher salience are executed first
‚ÄĘConflict resolution strategy applies between rules of the same 
salience
‚ÄĘIf salience and the conflict resolution strategy can‚Äôt 
determine which rule is to be executed next, a rule is chosen 
at random from the most highly ranked rules

Conflict Resolution Strategies
‚ÄĘDepth-first: newly activated rules placed above other rules in the

‚ÄĘBreadth-first: newly activated rules placed below other rules
‚ÄĘSpecificity: rules ordered by the number of conditions in the LHS

‚ÄĘLeast recently fired: fire the rule that was last fired the longest time

‚ÄĘRefraction: don‚Äôt fire a rule unless the WM patterns that match its

‚ÄĘRecency: rules ordered by the timestamps on the facts that match

 i) O-Start, a state in which O?s preconditions are satisfied.

‚ÄĘ C) If
 (First-Part <------ MEA (CURRENT, O-START)
 And

 Success and return the result of combining FIRST-PART, O, and LASTPART.

Adaptive learning
‚ÄĘ Adaptive machine learning algorithms are the machine learning 
models, where the changes in the environment help in selecting the 
algorithm or learning method.
‚ÄĘ The adaptive learning solves some of the complex problems for 
which a single learning method is not enough.
‚ÄĘ This method is even more appropriate when the environment is 
continuously changing and real time response is expected.

Multi-Agent System
‚ÄĘ A system with multiple autonomous 
entities, with distributed information, 
computational ability, and possibly 
divergent interests.
‚ÄĘ A multiagent system is one that 
consists of a number of agents, which 
interact with one-another
‚ÄĘ In the most general case, agents will be 
acting on behalf of users with different 
goals and motivations
‚ÄĘ To successfully interact, they will 
require the ability to cooperate, 
coordinate, and negotiate with each 
other, much as people do

Distributed learning
‚ÄĘ In distributed learning the task of learning is distributed.
‚ÄĘ Need for distributed learning - Arises due to large data sets and 
time constraints.
‚ÄĘ More than one agent in different parts of the data set.
‚ÄĘ There will be distributed learning algorithms taken part in each 
partition to get the desired outcome, which would then be 
combined.
‚ÄĘ Multi agent systems can be thought of as a subset of distributed 
learning.

Speedup learning
‚ÄĘ Speedup learning typically deals with speeding up problem solving by 
effective use of problem solving experience.
‚ÄĘ Hence, prior problem solving experience is an input for speedup learning.
‚ÄĘ In this learning,
1. There is no interaction with the environment.
2. New problems cannot be solved.
‚ÄĘ So, speedup learning accelerates the process based on the previous 
experiences and prior observations.

Expert System
‚ÄĘ An expert system is a computer program that is designed to solve 
complex problems and to provide decision-making ability like a 
human expert.
‚ÄĘ It performs this by extracting knowledge from its knowledge base 
using the reasoning and inference rules according to the user 
queries.


[Page 53]
Characteristics of Expert System
‚ÄĘ High Performance: The expert system provides high performance for
solving any type of complex problem of a specific domain with high
efficiency and accuracy.
‚ÄĘ Understandable: It responds in a way that can be easily
understandable by the user. It can take input in human language and
provides the output in the same way.
‚ÄĘ Reliable: It is much reliable for generating an efficient and accurate
output.
‚ÄĘ Highly responsive: ES provides the result for any complex query
within a very short period of time.

[Page 54]
Components of Expert System


[Page 55]
Components of Expert System
1. User Interface
‚ÄĘ With the help of a user interface, the expert system interacts with the user, takes queries as 
an input in a readable format, and passes it to the inference engine.
‚ÄĘ After getting the response from the inference engine, it displays the output to the user. 
‚ÄĘ In other words, it is an interface that helps a non -expert user to communicate with the expert 
system to find a solution.
2. Inference Engine(Rules of Engine)
‚ÄĘ The inference engine is known as the brain of the expert system as it is the main processing 
unit of the system. 
‚ÄĘ It applies inference rules to the knowledge base to derive a conclusion or deduce new 
information.
‚ÄĘ It helps in deriving an error -free solution of queries asked by the user.
3. Knowledge Base
‚ÄĘ The knowledgebase is a type of storage that stores knowledge acquired from the different experts
of the particular domain.
‚ÄĘ It is considered as big storage of knowledge.
‚ÄĘ The more the knowledge base, the more precise will be the Expert System.

Explanation Facility
‚ÄĘ Allows a user to understand how the 
expert system arrived at certain 
conclusions or results.
‚ÄĘ For example: it allows a doctor to find out 
the logic or rationale of the diagnosis 
made by a medical expert system.
Inference Engine
‚ÄĘ Seeks information and relationships from 
the knowledge base and provides 
answers, predictions, and suggestions the

‚ÄĘ Stores all relevant information, data,

the expert system.
‚ÄĘ It uses rules, If-then statements, fuzzy 
logic
Knowledge acquisition facility
‚ÄĘ Provide convenient and efficient means of 
capturing and storing all the components 
of the knowledge base.
‚ÄĘ Acts as an interface between experts and 
the knowledge base.

‚ÄĘ The main purpose of the user interface is 
to make the development and use of an 
expert system easier for users and 
decision makers
